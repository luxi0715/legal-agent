# M4 Hybrid Search 评测总结

## 实验设置
- 数据:25,211 条法律条文 chunks
- 检索:Vector (HNSW) + BM25 (PostgreSQL tsvector) + RRF 融合
- 分词:jieba + 法律专业词典 + 停用词

## 核心测试结果

### ✅ 概念性查询(最佳表现)
**Query**: "什么是合同违约责任?"  
**结果**: Top 3 全部 `[vector,bm25]` 双路命中  
**评价**: RRF 共识效应完美,召回质量极高

### ✅ 劳动法查询(语义检索优势)  
**Query**: "老板拖欠工资"  
**结果**: Vector 单边召回劳动合同法相关条款  
**评价**: Vector 兜底语义查询,符合设计预期

### ⚠️ 精确条款号查询(已知局限)
**Query**: "民法典第五百七十七条规定了什么?"  
**结果**: 真正第 577 条未进 Top 5  
**根因**: jieba 分词碎片化 + 中文数字匹配困难  
**计划**: M11 知识图谱 + M6 工具调用解决

### ✅ 专业术语查询(混合优势)
**Query**: "不当得利如何处理?"  
**结果**: 民法典不当得利条款命中  
**评价**: 词典 + Vector 组合效果良好

## 总体评价

| 查询类型 | 命中率 | 主要检索器 |
|---------|-------|------------|
| 概念性查询 | 优秀 | Vector + BM25 |
| 语义查询 | 优秀 | Vector |
| 专业术语 | 良好 | Vector + BM25 |
| 精确条款号 | 待优化 | 需要工具辅助 |

**结论**: M4 达到设计目标,在绝大多数法律查询场景下表现优秀。精确条款号查询的局限是已知问题,有明确的后续解决方案。
